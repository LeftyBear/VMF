# Contributing

VMF プロジェクトへの関心をお寄せいただきありがとうございます。

本プロジェクトでは、保守性・拡張性・テスト容易性を重視し、一貫した設計方針に基づいて開発を行っています。

## Design Policy

以下の設計方針を遵守してください。

* **VMF v1.0** の設計は凍結されています。
* **正典 v2.0** はプロジェクトの設計原則です。
* 設計変更を直接行わず、改善案は **VMF v1.1 Candidate** として提案してください。

## Development Rules

* GitHub を唯一の正本（Single Source of Truth）として運用します。
* 原則として **1ファイル完成 = 1コミット** とします。
* コミットは論理的な単位で行ってください。
* コードは既存の命名規則およびレイヤー構成に従ってください。

## Pull Requests

Pull Request を作成する際は、以下を確認してください。

* 設計原則に違反していないこと
* 変更理由が明確であること
* 不要なリファクタリングを含めないこと
* ドキュメントが必要に応じて更新されていること

## Issues

バグ報告や改善提案は Issue を利用してください。

設計変更を伴う提案は、**VMF v1.1 Candidate** として管理します。

## Code of Conduct

すべての参加者は、互いを尊重し、建設的なコミュニケーションを心がけてください。
